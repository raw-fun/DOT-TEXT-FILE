১.  নির্বাহী সারসংক্ষেপ (EXECUTIVE SUMMARY)
================================================================================

প্রজেক্ট নাম: AutoPrompt-X (AI Interaction Automator)
ট্যাগলাইন: "Codeless AI Automation - Speak Your Intent, Automate Your Work"

AutoPrompt-X হলো একটি বিপ্লবী Conversational DevOps সিস্টেম যা PowerShell 
স্ক্রিপ্টিং এবং Gemini AI-এর সমন্বয়ে তৈরি। এটি ব্যবহারকারীদের সাধারণ ভাষায় 
(বাংলা/ইংরেজি) নির্দেশ দেওয়ার মাধ্যমে জটিল AI অটোমেশন কাজ সম্পাদন করতে সক্ষম।

মূল উদ্ভাবন: 
- Natural Language to Automation Logic Conversion
- Zero Coding Required - শুধু বলুন কী চান
- Intelligent Loop Management with Anti-Ban Strategy
- Universal AI Platform Support (DeepSeek, ChatGPT, Claude, Gemini)
- Automatic Data Collection & Storage

প্রধান সমস্যা সমাধান: 
✓ হাজার বার একই প্রম্পট টাইপ করার যন্ত্রণা থেকে মুক্তি
✓ ঘণ্টার পর ঘণ্টা মনিটরের সামনে বসে থাকার প্রয়োজন নেই
✓ বড় ডেটাসেট জেনারেশন এখন সম্পূর্ণ স্বয়ংক্রিয়
✓ উৎপাদনশীলতা বৃদ্ধি ৫০০%+

মূল প্রয়োগক্ষেত্র: 
✓ বড় আকারের AI কন্টেন্ট জেনারেশন (১০০০+ আইডিয়া, আর্টিকেল, কোড)
✓ রিসার্চ ডেটা সংগ্রহ ও বিশ্লেষণ
✓ প্রম্পট চেইনিং ও মাল্টি-স্টেপ AI ওয়ার্কফ্লো
✓ A/B টেস্টিং ও প্রম্পট ইঞ্জিনিয়ারিং

২. প্রজেক্টের দৃষ্টিভঙ্গি ও উদ্দেশ্য (VISION & OBJECTIVES)
================================================================================

২. ১ দৃষ্টিভঙ্গি (Vision)
--------------------------
"একটি এমন বুদ্ধিমান অটোমেশন সিস্টেম তৈরি করা যেখানে ব্যবহারকারী শুধুমাত্র 
সাধারণ ভাষায় তার উদ্দেশ্য বলবেন এবং AI স্বয়ংক্রিয়ভাবে সেই কাজ সম্পন্ন 
করে দেবে—কোনো প্রোগ্রামিং বা কারিগরি জ্ঞান ছাড়াই।"

২.২ মূল উদ্দেশ্য (Primary Objectives)
---------------------------------------
[OBJ-001] ন্যাচারাল ল্যাঙ্গুয়েজ টু অটোমেশন
          → ইউজারের বাংলা/ইংরেজি কমান্ডকে সরাসরি এক্সিকিউটেবল কোডে রূপান্তর
          → কোনো প্রোগ্রামিং জ্ঞানের প্রয়োজন নেই

[OBJ-002] ইন্টেলিজেন্ট অটোমেশন ইঞ্জিন
          → Gemini AI ব্যবহার করে স্মার্ট প্ল্যানিং ও এক্সিকিউশন
          → Dynamic parameter extraction (count, delay, content)

[OBJ-003] মাল্টি-প্ল্যাটফর্ম AI সাপোর্ট
          → DeepSeek, ChatGPT, Claude, Gemini সব প্ল্যাটফর্মে কাজ করবে
          → Unified interface ব্যবহার করে বিভিন্ন AI-এ একই কমান্ড

[OBJ-004] রেট-লিমিট অ্যাওয়্যার অটোমেশন
          → বুদ্ধিমত্তার সাথে delay manage করে ban এড়ানো
          → Adaptive timing based on platform response

[OBJ-005] স্বয়ংক্রিয় ডেটা ম্যানেজমেন্ট
          → প্রতিটি রেসপন্স স্বয়ংক্রিয়ভাবে সংরক্ষণ
          → Structured output (TXT, CSV, JSON) সাপোর্ট

৩. সমস্যা বিবৃতি (PROBLEM STATEMENT)
================================================================================

৩.১ বর্তমান চ্যালেঞ্জসমূহ
---------------------------
[P-001] পুনরাবৃত্তিমূলক ম্যানুয়াল কাজ
        → AI থেকে বড় ডেটাসেট সংগ্রহে হাজার বার একই প্রম্পট টাইপ করতে হয়
        → উদাহরণ: ৫০০০ আইডিয়া জেনারেট করতে ৫০০০ বার "NEXT" লিখতে হয়
        → সময় ব্যয়:  প্রতি সেশনে ৪-৮ ঘণ্টা নিরবচ্ছিন্ন কাজ

[P-002] মনিটরিং এর বোঝা
        → পুরো প্রসেস চলাকালীন সময় কম্পিউটারের সামনে বসে থাকতে হয়
        → যদি ইন্টারনেট বিচ্ছিন্ন হয় বা সেশন expire হয় তাহলে সব ডেটা হারিয়ে যায়
        → Multitasking করার সুযোগ নেই

[P-003] কোডিং ব্যারিয়ার
        → বর্তমান অটোমেশন টুলগুলো ব্যবহার করতে programming দক্ষতা লাগে
        → সাধারণ ব্যবহারকারী complex script লিখতে পারেন না
        → প্রতিটি নতুন কাজের জন্য নতুন করে কোড লিখতে হয়

[P-004] প্ল্যাটফর্ম নির্ভরতা
        → একটি AI প্ল্যাটফর্মের জন্য লেখা স্ক্রিপ্ট অন্যটিতে কাজ করে না
        → প্রতিটি প্ল্যাটফর্মের জন্য আলাদা টুল দরকার হয়

[P-005] ডেটা সংগ্রহে অনিশ্চয়তা
        → Response manually copy-paste করতে গিয়ে ভুল হয়
        → ডেটা সংগঠিত ও সংরক্ষণে অনেক সময় নষ্ট হয়
        → Data loss এর ঝুঁকি থাকে

৩.২ প্রভাব বিশ্লেষণ
--------------------
- উৎপাদনশীলতা হ্রাস: ৭০-৮০% সময় শুধু repetitive কাজে নষ্ট
- মানসিক চাপ: একঘেয়ে কাজে মনোযোগ ও motivation কমে যায়
- সুযোগ ব্যয়: যে সময়ে সৃজনশীল কাজ করা যেত তা নষ্ট হয়
- স্কেলেবিলিটি সীমা: বড় প্রজেক্টে manual process অব্যবহারিক

৪.১ সলিউশন ওভারভিউ
---------------------
AutoPrompt-X একটি দ্বি-স্তরীয় বুদ্ধিমান সমাধান প্রদান করে: 

[স্তর ১] Natural Language Interface Layer
         → PowerShell-based Interactive Terminal
         → Bangla/English Command Recognition
         → User Intent Extraction Engine

[স্তর ২] AI-Powered Automation Planning Layer
         → Gemini AI Integration for Command Parsing
         → Intelligent Parameter Extraction
         → Executable Script Generation
         → Multi-Platform Automation Engine

কাজের প্রবাহ (Simplified):
ইউজার বলে → Gemini বুঝে → PowerShell করে → AI রেসপন্স আসে → সংরক্ষণ হয়

৪.২ মূল পার্থক্যকারী বৈশিষ্ট্য (Differentiators)
--------------------------------------------------
✓ Codeless Operation - শুধু সাধারণ ভাষায় বলুন
✓ AI-Driven Planning - Gemini স্বয়ংক্রিয় পরিকল্পনা তৈরি করে
✓ Universal Platform Support - সব AI-এ একই টুল
✓ Smart Delay Management - Ban/Rate-limit এড়াতে intelligent timing
✓ Auto Data Collection - সব রেসপন্স স্বয়ংক্রিয়ভাবে সেভ
✓ Resumable Sessions - ইন্টারাপশনের পর পুনরায় শুরু
✓ Zero Installation - শুধু . ps1 ফাইল রান করুন

৫.৩ ডেটা ফ্লো বিস্তারিত
--------------------------

[USER INPUT]
   "৫০০ বার NEXT লেখো ১ মিনিট ৪০ সেকেন্ড পর পর DeepSeek-এ"
   │
   ├─→ [NLP Processor] 
   │   ├─ ভাষা শনাক্তকরণ (বাংলা)
   │   ├─ সংখ্যা নিষ্কাশন:  500, 100 seconds
   │   ├─ ক্রিয়া শনাক্তকরণ:  "লেখো" → send_message
   │   ├─ কন্টেন্ট:  "NEXT"
   │   └─ টার্গেট:  "DeepSeek"
   │
   ├─→ [Gemini AI Processing]
   │   ├─ Intent Validation
   │   ├─ Parameter Extraction & Structuring
   │   ├─ Safety Check (Rate Limit Analysis)
   │   └─ JSON Plan Generation: 
   │       {
   │         "platform": "DeepSeek",
   │         "action": "send_message",
   │         "message_content": "NEXT",

৯.১ API Key নিরাপত্তা
-----------------------

【এনক্রিপশন】
├─ Gemini API Key সংরক্ষণ
│   ├─ AES-256 encryption ব্যবহার করে encrypt করা
│   ├─ Windows DPAPI (Data Protection API) integration
│   └─ User-specific encryption (machine-bound)
│
├─ First-time Setup
│   When user first runs: 
│   1.  Prompt for Gemini API Key
│   2.  Encrypt using ConvertTo-SecureString
│   3. Save to Config/api-keys.json (encrypted)
│   4. Original key immediately cleared from memory
│
└─ Runtime Usage
    1.  Decrypt only when needed
    2. Use in-memory only, never log
    3. Clear from memory after use

Code Example:
```powershell
# Save API Key (First time)
$apiKey = Read-Host "Enter Gemini API Key" -AsSecureString
$encryptedKey = ConvertFrom-SecureString $apiKey
Set-Content "Config/api-keys.json" -Value $encryptedKey

# Load API Key (Runtime)
$encryptedKey = Get-Content "Config/api-keys.json"
$secureKey = ConvertTo-SecureString $encryptedKey
$apiKey = [System.Runtime.InteropServices.Marshal]:: PtrToStringAuto(
    [System.Runtime.InteropServices.Marshal]::SecureStringToBSTR($secureKey)
)
```
